## What is Weiroll?

Weiroll is a flexible execution system designed to enable **trust-minimized and gas-efficient scripting** on Ethereum. It provides a powerful way to encode and execute complex sequences of operations on-chain in a deterministic and verifiable manner.

At its core, Weiroll uses a command/state execution model that allows you to:
- Encode multiple operations into a compact format
- Execute them atomically in a single transaction
- Pass state between operations efficiently
- Minimize gas costs through optimized encoding

<Info>
  Weiroll was originally created by the [weiroll project](https://github.com/weiroll/weiroll) and further enhanced by [Enso Finance](https://github.com/enso-finance/enso-weiroll).
</Info>

## The Problem Weiroll Solves

Traditional smart contract interactions often require:
- Multiple transactions for complex operations
- Custom contract code for each specific workflow
- High gas costs from repeated external calls
- Complex state management across transactions

Weiroll addresses these challenges by providing a **scripting layer** that lets you compose operations dynamically without deploying new contracts for each workflow.

## DeployableVM: A Simple Wrapper

DeployableVM is a lightweight wrapper around Enso Finance's audited Weiroll VM that makes it easy to integrate Weiroll functionality into any contract. The key difference is in how it's designed to be used:

<Note>
  **DeployableVM vs. Raw Weiroll VM**
  
  - **Raw Weiroll VM**: Requires integration into your contract's inheritance hierarchy
  - **DeployableVM**: Deployed once, used anywhere via `delegatecall`
  - **Security**: Enforces delegatecall-only execution for safety
</Note>

### Key Features

1. **Deploy Once, Use Everywhere**: Deploy a single instance and call it from any contract
2. **Delegatecall Pattern**: Executes in your contract's context, accessing your storage and balance
3. **Immutable Security**: Uses the `WEIROLL_SINGLETON` pattern to prevent direct calls
4. **Battle-Tested**: Built on Enso Finance's audited implementation

## Command/State Execution Model

Weiroll uses a unique two-part execution model:

### Commands
An array of `bytes32` values that encode:
- Target contract addresses
- Function selectors
- Input/output mappings
- Flags for call behavior

### State
An array of `bytes[]` that stores:
- Initial input values
- Intermediate results between operations
- Final return values

This separation allows for:
- **Compact encoding**: Commands are tightly packed
- **Efficient execution**: State is reused across operations
- **Composability**: Outputs feed into subsequent inputs

## Getting Started

To use DeployableVM in your contract:

1. Deploy or reference an existing DeployableVM instance
2. Encode your operations into Weiroll commands and state
3. Call the VM via `delegatecall` from your contract
4. Handle the returned state array

## Prerequisites

Before installing DeployableVM, ensure you have:

- [Foundry](https://book.getfoundry.sh/getting-started/installation) installed
- A Solidity project initialized with `forge init`
- Solidity compiler version 0.8.27 or higher

<Warning>
  For deterministic deployment compatibility, use the specific Foundry toolchain version:
  ```bash
  foundryup -C cb9dfae298fe0b5a5cdef2536955f50b8c7f0bf5
  ```
  This ensures consistent deployment addresses across different environments.
</Warning>

## Installation Methods

<Tabs>
  <Tab title="Use Existing Deployment">
    The simplest approach is to use the already-deployed DeployableVM contract available on multiple chains. No installation required!

    See the [Deployment Addresses](#deployment-addresses) section below for the contract address on your target chain.
  </Tab>
  <Tab title="Foundry (Recommended)">
    Install the DeployableVM contract and its dependencies in your Foundry project:

    <Steps>
      <Step title="Add as Forge Dependency">
        Install the contract directly from the GitHub repository:
        ```bash
        forge install cowprotocol/weiroll
        ```
      </Step>

      <Step title="Update Remappings">
        Add the remapping to your `foundry.toml` or `remappings.txt`:
        ```toml
        @cowprotocol/weiroll/=lib/weiroll/
        ```
      </Step>

      <Step title="Import in Your Contract">
        Import the DeployableVM contract:
        ```solidity
        import {DeployableVM} from "@cowprotocol/weiroll/src/DeployableVM.sol";
        ```
      </Step>
    </Steps>
  </Tab>
  <Tab title="From Source">
    Clone and build from source for development or custom deployments:

    <Steps>
      <Step title="Clone Repository">
        ```bash
        git clone https://github.com/cowprotocol/weiroll.git
        cd weiroll
        ```
      </Step>

      <Step title="Install Dependencies">
        ```bash
        forge install
        ```
      </Step>

      <Step title="Build">
        ```bash
        forge build
        ```
      </Step>

      <Step title="Run Tests">
        Verify the installation:
        ```bash
        forge test
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Compiler Configuration

To match the production deployment settings and achieve optimal gas efficiency, configure your `foundry.toml`:

```toml
[profile.default]
src = "src"
out = "out"
libs = ["lib"]

# Compiler settings
via_ir = true
optimizer = true
optimizer_runs = 1000000
solc = "0.8.27"

[fmt]
sort_imports = true
```

<Info>
  **Why these settings?**
  - `via_ir = true`: Enables advanced optimization through the intermediate representation compiler
  - `optimizer_runs = 1000000`: Optimizes for contracts that will be called frequently, reducing execution costs
  - `solc = "0.8.27"`: Matches the production deployment version
</Info>

## Custom Deployment

If you need to deploy your own instance of DeployableVM:

<Steps>
  <Step title="Set Environment Variables">
    ```bash
    export ETHERSCAN_API_KEY='your-api-key-here'
    export PK='your-private-key'
    export ETH_RPC_URL='https://your-rpc-url.example.com'
    ```
  </Step>

  <Step title="Verify Deterministic Address">
    Preview the deployment without broadcasting:
    ```bash
    forge script 'script/DeployableVM.s.sol:DeployableVMScript' \
      -vvvv \
      --rpc-url "$ETH_RPC_URL" \
      --private-key "$PK"
    ```
  </Step>

  <Step title="Deploy and Verify">
    Deploy the contract and verify on the block explorer:
    ```bash
    forge script 'script/DeployableVM.s.sol:DeployableVMScript' \
      -vvvv \
      --rpc-url "$ETH_RPC_URL" \
      --private-key "$PK" \
      --verify \
      --broadcast
    ```
  </Step>
</Steps>

<Warning>
  Ensure you're using the correct Foundry version (see Prerequisites) to generate the expected deterministic address.
</Warning>

## Verification

To verify your installation is working correctly:

```bash
forge test
```

You should see successful test output confirming:
- The contract reverts when called directly (not via delegatecall)
- The contract executes successfully when called via delegatecall

## Overview

The DeployableVM contract is deployed at deterministic addresses across multiple EVM-compatible chains. All deployments use the same bytecode and are deployed using CREATE2 for address consistency.

<Note>
All addresses are verified deployments of the DeployableVM contract. You can use these addresses to interact with the Weiroll VM via delegatecall from your contracts.
</Note>

## Usage

To use the DeployableVM in your contract, reference the appropriate address for your target chain:

```solidity
contract MyContract {
    address constant WEIROLL_VM = 0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963;
    
    function executeWeiroll(
        bytes32[] calldata commands,
        bytes[] memory state
    ) external payable returns (bytes[] memory) {
        (bool success, bytes memory result) = WEIROLL_VM.delegatecall(
            abi.encodeWithSelector(
                DeployableVM.execute.selector,
                commands,
                state
            )
        );
        
        require(success, "Weiroll execution failed");
        return abi.decode(result, (bytes[]));
    }
}
```

## Verification

You can verify these deployments by:

1. Checking the transaction hash on the respective block explorer
2. Verifying the contract bytecode matches the DeployableVM source
3. Confirming the contract is verified on Etherscan or equivalent explorers

<Warning>
Always verify the contract address on the block explorer for your target chain before integrating. While these addresses are accurate as of deployment, always confirm you're interacting with the correct contract.
</Warning>

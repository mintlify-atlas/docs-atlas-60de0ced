## Overview

DeployableVM is deployed at the same deterministic address across all supported EVM chains using CREATE2 with an empty salt. This ensures consistent integration regardless of which chain you're targeting.

<Info>
All deployments use the deterministic address: `0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963`
</Info>

## Integration Examples

You can reference DeployableVM in your contracts using the deterministic address:

<CodeGroup>
```solidity Hardcoded Reference
contract YourContract {
    address private constant WEIROLL_VM = 0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963;

    function executeWeiroll(
        bytes32[] calldata commands,
        bytes[] memory state
    ) external payable returns (bytes[] memory) {
        (bool success, bytes memory result) = WEIROLL_VM.delegatecall(
            abi.encodeWithSignature(
                "execute(bytes32[],bytes[])",
                commands,
                state
            )
        );
        require(success, "Execution failed");
        return abi.decode(result, (bytes[]));
    }
}
```

```solidity Constructor Injection
contract YourContract {
    address public immutable weirollVM;

    constructor() {
        // Same address on all chains
        weirollVM = 0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963;
    }

    function executeWeiroll(
        bytes32[] calldata commands,
        bytes[] memory state
    ) external payable returns (bytes[] memory) {
        (bool success, bytes memory result) = weirollVM.delegatecall(
            abi.encodeWithSignature(
                "execute(bytes32[],bytes[])",
                commands,
                state
            )
        );
        require(success, "Execution failed");
        return abi.decode(result, (bytes[]));
    }
}
```

```solidity Multi-Chain Factory
contract MultiChainFactory {
    address private constant WEIROLL_VM = 0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963;

    function deployContract() external returns (address) {
        // Deploy contracts that use the same Weiroll VM
        // across all supported chains
        return address(new YourContract(WEIROLL_VM));
    }
}
```
</CodeGroup>

## Verifying Deployments

### Via Block Explorer

You can verify any deployment by visiting the block explorer link and checking:

1. The contract bytecode matches other chains
2. The deployment transaction hash matches `networks.json`
3. The contract is verified and shows the correct source code

### Via Cast (Command Line)

Verify the bytecode is identical across chains:

```bash
# Get bytecode from Ethereum Mainnet
cast code 0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963 \
  --rpc-url https://eth.llamarpc.com

# Get bytecode from Arbitrum
cast code 0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963 \
  --rpc-url https://arb1.arbitrum.io/rpc

# Compare - they should be identical
```

### Via Script

Automate verification across multiple chains:

```bash
#!/bin/bash

ADDRESS="0x9585c3062Df1C247d5E373Cfca9167F7dC2b5963"

chains=(
  "https://eth.llamarpc.com"
  "https://arb1.arbitrum.io/rpc"
  "https://polygon-rpc.com"
  "https://mainnet.base.org"
)

for rpc in "${chains[@]}"; do
  echo "Checking $rpc"
  cast code $ADDRESS --rpc-url $rpc | head -c 100
  echo ""
done
```

## Request New Chain Deployment

To request DeployableVM deployment on a new EVM chain:

1. Open an issue on the [GitHub repository](https://github.com/cowprotocol/weiroll/issues)
2. Provide the chain name, chain ID, and RPC endpoint
3. Confirm the chain is EVM-compatible and supports CREATE2
4. Community or core team will handle the deployment

Or deploy it yourself following the [deployment guide](/guides/deployment) and submit a PR with the updated `networks.json` file.
